SOLUCION PARA EL MENSAJE "SIN CONEXION"

URL configurada:
https://script.google.com/macros/s/AKfycbxFXmuRuyXFKMMurZ3_wzkyjt6bRb_pWhBgvzDbhA6rgUYeCFHoxNfWc-N2D9FZF6C7/exec

1. Abra la URL anterior en una ventana de incógnito.
2. Agregue al final:
   ?action=ping
3. El resultado correcto debe ser un JSON parecido a:
   {"ok":true,"nombre":"...","id":"..."}
4. Si aparece una pantalla de inicio de sesión, vuelva a implementar Apps Script:
   - Implementar > Administrar implementaciones.
   - Editar o crear Nueva implementación.
   - Tipo: Aplicación web.
   - Ejecutar como: usted.
   - Acceso: Cualquier usuario o cualquier usuario con el enlace, según permita la cuenta institucional.
   - Seleccione una VERSIÓN NUEVA del código.
5. Use siempre la URL que termina en /exec, no /dev.
6. Para evitar restricciones del navegador, no abra index.html solamente con doble clic.
   Ejecútelo desde un servidor local, por ejemplo:

   Windows, dentro de la carpeta del proyecto:
   python -m http.server 8000

   Luego abra:
   http://localhost:8000

La versión corregida realiza dos pruebas:
- Lectura mediante JSONP.
- Escritura de respaldo mediante POST.

Si el navegador impide leer la respuesta pero permite enviar datos, el sistema mostrará:
"Conexión de escritura activa" en lugar de un falso mensaje de desconexión.

No se modificó la estructura funcional del sistema ni las hojas de Google Sheets.
